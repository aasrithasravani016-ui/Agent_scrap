We built two separate features. They both answer questions about network switches, but they look up different things in different places.

Version 1   Specifications

Someone asks: "What are the specs of the Cisco Catalyst 9300?" — things like port count, speed, power, layer, etc.

Step 1 — Look in our own saved file first

We keep a small local database called switches.db (it's a SQLite file — basically a tiny self-contained Excel for code). If the model is there, we answer in about 1 millisecond. Done.

Step 2 — If not saved, go to the vendor's website

We open the vendor's public product page directly — for example mikrotik.com, cisco.com, arista.com, netgear.com, nvidia.com, ui.com — using the Python requests library, read the HTML, pull out the specs with BeautifulSoup (HTML parser) and pdfplumber / PyMuPDF (PDF parsers), and save them. Next time it's instant.

Step 3 — If the vendor blocks us, do a web search

Some vendors (Aruba, Dell, Juniper) refuse our agent at the door (their security system, Akamai, returns a "403 Forbidden" error). So instead we do a normal web search through Startpage, then Mojeek, then DuckDuckGo as a last resort. We open the first good result — usually the official datasheet on the vendor's own site or a reseller's page — and read it the same way.

Why three? Startpage and Mojeek are independent search engines that don't track us and rarely block automated requests. DuckDuckGo started rate-limiting us, so we moved it to the bottom of the list.

Step 4 — If we still can't get full specs, give the link

If the datasheet is messy (image-based table, weird PDF, JavaScript page), we don't make up numbers. We honestly say: "Here is the official datasheet link and the few features we could read — please open the link for the rest."

Version 2   Firmware & Security

Someone asks: "What's the latest firmware for Cisco IOS-XE? Any known security issues?"

Step 1 — Look in our firmware list first

For vendors who publish their firmware notes openly, we already collected 127 versions directly from their public download pages:

  • MikroTik RouterOS from upgrade.mikrotik.com/routeros (plain-text changelog files).
  • NVIDIA Cumulus Linux from docs.nvidia.com + GitHub release notes.
  • Ubiquiti UniFi from their firmware API fw-update.ui.com/api/firmware + community.ui.com/releases.

We show the latest version, the release date, and what was fixed.

Step 2 — If the vendor hides firmware behind a login

Big vendors like Cisco, Juniper, HPE Aruba, Dell put their firmware download pages behind a customer login portal (software.cisco.com, support.juniper.net, asp.arubanetworks.com, dell.com/support). We don't have logins for these, so we can't reach them. Instead of giving up, we do the next best thing:

Step 3 — Pull public security issues from NIST NVD

NIST NVD stands for the U.S. National Institute of Standards and Technology — National Vulnerability Database. It is the U.S. government's free, public security database that lists every known security bug (called a "CVE") for every vendor, including Cisco, Juniper, Aruba and the rest. No login, no fee.

We call its API at services.nvd.nist.gov/rest/json/cves/2.0 and store the results in our database. So far we have 2,728 issues across 12 vendors. For each one we show:

  • Which security bug affects this vendor (the CVE number, like a license-plate ID).
  • How serious it is — the CVSS score (Critical / High / Medium / Low).
  • Which firmware version finally fixed it (so the user knows what to upgrade to).
  • A red flag for the 68 bugs that the U.S. cybersecurity agency CISA says are being actively attacked right now — pulled from CISA's Known Exploited Vulnerabilities (KEV) catalog.

Step 4 — Be honest about what we don't have

We tell the user clearly: "We don't have the latest firmware version number for this vendor because their downloads need a login. But here are the public security issues you should know about, and here is the official login link if you want the full release notes."

Problems we hit, and how we solved them

The problemWhat we did
Big vendors (HPE Aruba, Dell, Juniper) use a service called Akamai that blocks automated visitors — we get a "403 Forbidden" error. For specs, we fall back to Startpage / Mojeek web search and read the first good result. For firmware, we use the free NIST NVD security database instead.
TP-Link product pages load empty because they need a real browser to "draw" them (JavaScript-rendered). Read their sitemap.xml (a hidden text list of every page on tp-link.com), then fetch the static product pages directly.
DuckDuckGo search started refusing our requests (returning HTTP 202). Added Startpage and Mojeek as the first two engines. DuckDuckGo is now only a last resort.
NIST NVD only allows 5 requests every 30 seconds without an API key. Added a 6.5-second wait between requests, plus an option to top-up one vendor at a time (e.g. --nvd-vendor cisco).
Our Streamlit web app crashed under multiple users because SQLite blocks cross-thread access by default. Opened the database with check_same_thread=False and cached the agent with @st.cache_resource.
Rebuilding the database from seed used to wipe out all 237 scraped models. Switched to in-place migrations (ALTER TABLE / CREATE TABLE IF NOT EXISTS) — like editing a spreadsheet instead of deleting it.

What we honestly cannot do (yet)

  • We can't get the actual latest firmware version for Cisco, Juniper, HPE Aruba, Dell — those downloads are behind a login on software.cisco.com etc. We can show the CVE list, but not the version number.
  • We can't read messy PDF datasheets perfectly. Without an LLM (AI) to "read" images and weird layouts, we sometimes give only the link and a few features.
  • No single complete list of every switch in the world exists. We stitch many sources together. Some old or niche models will always be missing.
  • Chinese vendors (Huawei, H3C, Ruijie) have much richer data on their Chinese pages than the English pages we read.

What's missing / what to do next

  • About 115 vendors we know about have no auto-fetch set up yet (mostly small / industrial / niche brands).
  • No firmware version data yet for Cisco, Juniper, Arista, Aruba — only CVE data from NVD.
  • No "end-of-life" / "end-of-support" dates yet.
  • Nothing runs on a schedule — we re-fetch by hand. Could be put on a daily auto-run (e.g. GitHub Actions, cron).
