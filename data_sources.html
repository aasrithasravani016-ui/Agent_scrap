How it works (in simple steps)

When someone asks the agent about a switch, it tries three places, in this order:

  1. Our own saved file — like a notebook we already filled in. Fastest.
  2. Vendor websites — if the answer isn't in the notebook, the agent goes to the maker's website (Cisco, MikroTik, etc.) and reads the public product page.
  3. Web search — if even the vendor page doesn't have it, the agent does a normal Google-style search and reads the first good result.

The whole thing runs at zero cost. No paid services, no AI charges. It just reads public web pages.

Problems we hit, and how we solved them

The problemWhat we did
Big vendors (Aruba, Dell, Juniper) block automated visitors from their websites. Our agent gets turned away at the door. We stopped trying to walk through their front door. Instead we pull the security data from the free U.S. government database, which lists the same issues without any login.
TP-Link's web pages load empty for our agent because they need a browser to "draw" the page. We found their sitemap (a hidden text list of every page on their site) and read it directly. No drawing needed.
The popular Google-style search engine we first used started refusing our requests after a few tries. We added two backup search engines. If the first one says no, we try the second, then the third.
The government security database limits us to 5 requests every 30 seconds. We added a small wait timer between requests so we never go over the limit.
The web app sometimes crashed when many people used it at once (database thread errors). Changed one database setting that allows it to be used by multiple users at the same time.
Rebuilding the database used to wipe out all our scraped data. Now we update the database in place instead of rebuilding from scratch — like editing a spreadsheet instead of deleting it and starting over.

What we honestly cannot do

  • We can't get full release notes from "members-only" websites. Cisco, Aruba, Juniper, Dell, and Extreme hide the detailed update notes behind a login screen. We can only see the security issue list, not the full notes.
  • We can't read messy PDF datasheets perfectly. Some vendors put their specs inside images or weird layouts. Without an AI to "read" them, we can only give the link and a few features.
  • There is no single complete list of every switch in the world. We are stitching together many sources. Some vendors and old models will always be missing.
  • Some Chinese vendors (Huawei, H3C, Ruijie) put much more detail on their Chinese pages than their English pages. We only read the English ones.

What's still missing / what to do next

Coverage gaps

  • No firmware data yet for Cisco, Juniper, Arista, Aruba — only security issues.
  • About 115 vendors we know about have no auto-fetch set up yet (mostly small / industrial / niche brands).
  • Huawei, H3C, Ruijie data is thin.

Quality gaps

  • Many saved switches only have a link to the datasheet, not all the numbers.
  • No "end-of-life" / "end-of-support" dates yet.
  • Nothing runs on a schedule — we re-fetch by hand.
